# RT-Thread 与云端联调说明

日期：2026-05-19

## 目录作用

- `xiaozhi-server`：FastAPI 后端。设备 WebSocket 入口是 `/xiaozhi/v1/`，前端 REST 接口是 `/api/sensor/*`、`/api/meow/*` 等。
- `xiaozhi-web`：Vue 3 前端。温湿度页读取 `/api/sensor/latest/history/stats`，猫叫页读取 `/api/meow/events/stats`。
- `Edgi_Talk_M55_XiaoZhi_2/Edgi_Talk_M55_XiaoZhi`：主 RT-Thread 小智固件工程。现在已接入温湿度与猫叫检测上报。
- `UDP`：独立 Wi-Fi/UDP 示例工程，主要用于参考网络通信和继电器控制，不是当前主链路。
- `key_camera`：独立串口摄像头 PTC06 示例工程，包含拍照、分片发送思路，不是当前温湿度/猫叫主链路。
- `模型-hachimitsu_detector_int8-no_mel.tflite`：猫叫模型文件，主固件里当前使用的是已转成 C 数组的 TFLite 模型头文件。

## 当前通信链路

设备连接：

```text
RT-Thread 固件
  -> ws://192.168.10.132:8000/xiaozhi/v1/
  -> Authorization: Bearer 12345678
  -> hello 握手
  -> 后端返回 session_id
```

温湿度上报：

```json
{
  "session_id": "后端返回的 session_id",
  "type": "env",
  "temp_c": 27.30,
  "humi_rh": 62.40,
  "ts": 123456
}
```

猫叫上报：

```json
{
  "session_id": "后端返回的 session_id",
  "type": "meow",
  "score": 0.93,
  "is_cat": true,
  "ts": 123456
}
```

后端入库后，前端会通过 REST 和 `/ws/dashboard` 实时看到更新。

## 固件改动点

- `applications/xiaozhi/env_ws_uploader.c/.h`：新增 AHT20 I2C 读取和温湿度 WebSocket 周期上报，默认 10 秒一次。
- `applications/xiaozhi/xiaozhi.cpp`：新增通用 `ws_send_text_locked()`，在 `hello` 成功后启动温湿度上报线程；同时接收后端 `meow_control` 开关命令。
- `applications/xiaozhi/wake_word/meow_detect_once.cpp`：每次猫叫推理后向云端发送 `meow` 消息。
- `applications/xiaozhi/xiaozhi.h`：默认云端地址改为当前电脑 WLAN IP `192.168.10.132`，并保留 `#ifndef XIAOZHI_HOST` 方便编译时覆盖。

## 后端改动点

- `app/routers/ws_device.py`：`env`/`sensor` 消息统一走温湿度解析器，兼容 `temp_c/humi_rh` 和旧字段。
- `app/services/sensor_pipeline.py`：补充 `humi_rh` 字段兼容。
- `tests/test_ws_device_ingest.py`：新增设备 WebSocket 入库测试，覆盖温湿度和猫叫。

## 联调步骤

1. 确认电脑和 RT-Thread 板子在同一局域网。
2. 启动后端：在 `xiaozhi-server` 目录运行 `.\.venv\Scripts\python.exe run.py`，端口 `8000`。
3. 启动前端：在 `xiaozhi-web` 目录运行 `npm run dev -- --host 0.0.0.0`，本机打开 `http://localhost:5173/`。
4. 重新编译并烧录主固件工程。
5. 串口日志应出现 WebSocket connected / hello response / env uploaded。
6. 打开前端温湿度页，应能看到真实 AHT20 上报数据；打开猫叫页，应能看到检测事件。

如果换了 Wi-Fi，先用 PowerShell 查看电脑 IPv4：

```powershell
Get-NetIPAddress -AddressFamily IPv4
```

然后把 `applications/xiaozhi/xiaozhi.h` 中的 `XIAOZHI_HOST` 改成新的电脑 WLAN IP，或在编译参数里定义 `XIAOZHI_HOST` 覆盖默认值。
